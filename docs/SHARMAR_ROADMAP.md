# SHARMAR — Product & Engineering Roadmap

---

## ФАЗА 0 — СТАБИЛИЗАЦИЯ (обязательная)

Цель: иметь надёжную, восстановимую, управляемую систему.

Задачи:
- аудит Docker / volumes / env
- схема бэкапов (DB + uploads + config)
- проверка восстановления
- фиксация production-процесса
- стабилизация Strapi (права, роли, публикации)

Критерий готовности:
- сайт переживает падение
- данные не теряются
- админка стабильна
- контент не пропадает

---

## ФАЗА 1 — ЯДРО ПРОДУКТА

Цель: финальная структура данных.

Задачи:
- финализация моделей (Boat, Brand, Location, Purpose, RatePlan, Owner)
- система ролей
- заявки
- модерация
- статусы лодок
- медиа-стратегия

Критерий готовности:
- можно безопасно наполнять сайт
- данные не требуют переделок

---

## ФАЗА 2 — МУЛЬТИЯЗЫЧНОСТЬ И AI

Цель: автоматизация без разрушения.

Задачи:
- стратегия локалей
- схема перевода
- AI-очередь переводов
- ручной контроль
- защита оригинала

Критерий готовности:
- можно массово наполнять сайт
- переводы не ломают SEO

---

## ФАЗА 3 — SEO-АРХИТЕКТУРА

Цель: органический рост.

Задачи:
- URL-структура
- программные SEO-страницы
- страницы городов
- страницы типов лодок
- фильтрация без дублей
- sitemap / hreflang / schema.org

Критерий готовности:
- сайт готов к индексации
- нет SEO-долгов

---

## ФАЗА 4 — ИНТЕРФЕЙС И КОНВЕРСИЯ

Цель: превратить трафик в заявки.

Задачи:
- UX
- карточки лодок
- формы заявок
- доверие
- скорость
- мобильная оптимизация

---

## ФАЗА 5 — ПРОДУКТ И РОСТ

Цель: масштабируемая платформа.

Задачи:
- аккаунты владельцев
- аналитика
- личные кабинеты
- платные размещения
- API
- партнёрства
